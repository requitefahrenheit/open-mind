## DESIGN SYSTEM: MONOPOLY CARD AESTHETIC
The app uses a Monopoly board game card aesthetic. ALL UI elements — Stream cards, Focus view, Triage cards, Canvas views, detail panels, Resurface prompts, Chat panels — must follow this pattern:
- Nodes are rendered as Monopoly-style property cards (colored header band by node type, title, content preview, metadata footer)
- The card metaphor is the core visual language of the entire app. Every view that displays a node should render it as a card.
- New features (Resurface, Canvas Chat, Chat With Selection, Structured Fields) must also use cards. Don't introduce new UI patterns that break the card metaphor.

## LAYOUT: RESIZABLE DIVIDER
The app should have a resizable divider between the card list/view area and the detail panel:
- Drag handle between the two panes
- User can resize the split to give more room to either the card list or the detail view
- Persist the divider position (localStorage is fine)
- Works on both desktop and mobile (on mobile, likely a top/bottom split or full-screen toggle instead)

## NAVIGATION: ARROW KEYS
Arrow key navigation through cards:
- Up/Down arrows move between cards in Stream, Triage, search results, and canvas node lists
- Currently selected card is highlighted; pressing Enter opens its detail
- Arrow keys should work in any view that presents a list of cards
- This is essential for keyboard-first workflow
